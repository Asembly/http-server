# HTTP‑сервер на Java

Текущая **nio** ветка сервера, это новое глобальное изменение в плане обработки данных, теперь 
сервер работает в неблокирующем режиме чтения и записи, что позволяет ему всего лишь на одном потоке обрабатывать до
10тыс запросов/секунду и более. Пока что не реализован полностью весь потенциал, т.к сервер работает всего лишь на одном
потоке, в последующих версиях количество потоков будет создаваться в зависимости от количества
ядер процессора на текущей машине где этот сервер будет запущен.

### Возможности

- Выдача статики
- Маршрутизация запросов.
- Парсинг HTTP.
- Конфигурационный файл
- Переход на неблокирующий подход
- **Reverse proxy** (в доработке).
- Cache:
  - ***Кэширование на данный момент реализовано на базовом уровне. Создан обычный Lazy Cache который кэширует каждый запрос без обновления и удаления.***
- LoadBalancer: 
  - ***Сейчас реализован простой Round Robin алгоритм, в последующих апдейтах будет добавлен health-check.***

### В разработке

- Более сложная логика кэширования и проксирования с помощью LoadBalancer
- Поддержка **HTTPS (TLS‑шифрование)**.

Проект учебный, API и архитектура могут меняться по мере разработки.
